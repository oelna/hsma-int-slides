Ein kurzer Exkurs über das Dateisystem

  • Dateien (Files) sind digitale Dokumente – zB. Fotos, Texte, Videos, PDFs.
  • Ein Verzeichnis (Ordner, "Directory") ist eine Sammlung von Dateien, eine Möglichkeit, mehrere Dateien zu gruppieren.
  • Das Dateisystem ist das Organisationssystem auf deinem Computer, das bestimmt, wie Dateien und Ordner strukturiert, benannt und gefunden werden können.

Dateien / Files

  • Eine Datei speichert Informationen, wie z.B. ein Foto, Musik, Text oder ein Video.
  • Jede Datei hat einen Namen und meist auch eine Dateiendung (z.B. .jpg, .txt, .pdf, .mp3), die angibt, um welchen Typ es sich handelt.
  • Eine Datei besteht aus einer Reihe von Bytes und einer Bezeichnung (Name + Speicherort, Pfad, Path)
    zB. /Users/Arno/Desktop/cat.jpg

Dateien / Files

  • Erstellt man eine neue Datei mit den gleichen Bytes wie einer anderen, aber einem anderen Speicherort, spricht man von einer Kopie (copy)
  • Ändert man nur den Speicherort einer Datei, sagt man die Datei wurde bewegt oder verschoben (moved)
  • Ändert man die Bytes, aber der Speicherort bleibt gleich, sagt man die Datei wurde bearbeitet oder geändert (changed, edited)

Verzeichnisse / Directories

  • Ein Verzeichnis ist ein Container, in dem du mehrere Dateien zusammenfassen kannst.
  • Es ist vergleichbar mit einem digitalen Aktenschrank oder Fotoalbum, in dem du Dinge thematisch gruppierst.
  • Du kannst Ordner verschachteln, also Ordner in anderen Ordnern ablegen, um eine klare Struktur zu schaffen.

Verzeichnisse / Directories

  • Gute Verzeichnis- und Dateinamen helfen dir, deine gespeicherten Inhalte später wieder zu finden
  • Die Ordner und Unterordner, die uns zum Inhalt einer bestimmten Datei bringen, bezeichnet man als Pfad (path)
  • Pfade können unter verschiedenen Betriebssystemen unterschiedlich aussehen:
    /Users/Arno/Desktop/cat.jpg macOS
    C:\Users\Arno\Desktop\cat.jpg Windows
    https://arnorichter.de/hsma/index.html Browser, URL

Dateisystem / File System

  • Das Dateisystem verwaltet alle Dateien und Verzeichnisse auf deinem Gerät.
  • Es legt fest, wo jede Datei gespeichert wird und wie sie gefunden werden kann – ähnlich wie ein Inhaltsverzeichnis in einem Buch.
  • Verbreitete Dateisysteme sind zB. APFS, NTFS, exFAT, EXT4

Dateisystem / File System

  • Manche Geräte, zB. Smartphones, Router, etc., erlauben keinen direkten Zugriff ihre Dateien und abstrahieren das Dateisystem.
  • Trotzdem funktionieren sie alle auf die gleiche Weise wie ein normaler Computer
  • Eine Suchfunktion ist ein schlechter Ersatz für ein gut geordnetes System von Dateinamen und Ordnern.

Dateinamen

Achte darauf, dass du in deinem Betriebssystem einstellst, dass es dir vollständige Dateinamen inkl. Dateiendung (Erweiterung, extension) anzeigt!

HTML-Seiten sind
Textdokumente.

HTML verwendet Elemente,
so genannte Tags.

Tags fungieren als Container und sagen etwas über die Information aus, die dazwischen steht.

Tags bestehen in der Regel immer aus einem Opening Tag und einem Closing Tag.

Der Opening Tag markiert den Beginn eines Inhaltsbereichs, der Closing Tag das Ende.

		
			<p>Hello World!</p>

Tags können ineinander verschachtelt werden. Dadurch erreicht man eine Baumstruktur ähnlich dem Dateisystem auf deinem Rechner.

Zur Übersicht rückt man dann oft zeilenweise ein.

Das kann man wahlweise mit Leerzeichen oder Tabulator-Zeichen tun. Man sollte nur konsistent bleiben.

Opening Tags können Attribute tragen.

		
			<html>

html Element

Opening Tags können Attribute tragen.

		
			<html lang="de">

Tag-Name Attribut-Name Attribut-Wert

Attribute beschreiben ein Element/Tag genauer.

Sie bestehen immer aus Paaren von Name="Wert"

Elemente können beliebig viele Attribute tragen:

		
			<img src="bild1.jpg" width="600">
